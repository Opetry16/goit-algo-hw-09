# Висновки

У ході розробки системи для касового апарату, яка визначає оптимальний спосіб видачі решти покупцеві, були реалізовані два алгоритми: жадібний та динамічне програмування. Обидва алгоритми мають свої переваги та обмеження, і їх ефективність визначалася на різних сценаріях.

## Жадібний алгоритм

Жадібний алгоритм дозволяє швидко знаходити рішення, вибираючи найбільш доступні номінали монет. Він ефективний при великих сумах та швидко видає решту. Однак, в певних ситуаціях може не гарантувати мінімальну кількість монет.

## Динамічне програмування

Алгоритм динамічного програмування, використовуючи таблицю dp, забезпечує точне знаходження мінімальної кількості монет для кожної суми. Він дозволяє отримати оптимальний результат, але може бути менш швидким при великих сумах порівняно із жадібним алгоритмом.

## Порівняння та висновки

Проведені експерименти показали, що жадібний алгоритм ефективний при великих сумах та швидко вирішує завдання видачі решти. З іншого боку, алгоритм динамічного програмування виявився більш точним і міг бути більш ефективним при мінімізації кількості монет для конкретних сум.

Загально кажучи, вибір між цими двома алгоритмами повинен залежати від конкретних вимог та обмежень системи. Жадібний алгоритм може бути оптимальним в ситуаціях, де швидкість важлива, а динамічне програмування - коли необхідно гарантувати мінімальну кількість монет.
